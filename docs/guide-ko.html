남는 태블릿을 PC 조작 패널로

Surecut Deck은 Windows PC에서 작은 서버를 실행하고 자기 화면을 직접 제공합니다. 태블릿은 웹 주소를 열기만 하면 됩니다. 설치할 앱도, 만들 계정도, 중간의 클라우드도 없습니다.

작동 방식

호스트는 당신의 PC알림 영역의 작은 프로그램이 로컬 서버를 띄우고 계속 실행합니다. 설치하는 것은 이것뿐입니다.
태블릿은 주소만 엽니다같은 네트워크의 어떤 브라우저든 됩니다. 홈 화면에 추가하면 앱처럼 동작합니다.
버튼은 진짜 입력을 보냅니다누르면 실제 Windows 키보드·마우스 이벤트가 되므로 어떤 프로그램이든 반응합니다.

세 단계, 한 번만.

연결하기

1

서버 실행

start.cmd를 실행하세요. 알림 영역에 아이콘이 나타나 계속 남아 있습니다. 언제든 눌러 페어링 정보를 보거나 편집기를 열 수 있습니다.

2

태블릿에서 주소 열기

서버 창에는 PC에 접근할 수 있는 모든 주소가 나옵니다. 그중 하나를 태블릿 브라우저에 입력한 뒤, 옆에 표시된 페어링 코드를 넣으세요.

3

앱처럼 만들기

⛶를 눌러 전체 화면으로 바꾸거나 브라우저의 홈 화면에 추가를 사용하세요. 전체 화면에서는 덱이 열려 있는 동안 태블릿도 잠들지 않습니다.

기본

버튼은 키를 누르는 것 이상을 합니다

연필을 눌러 편집 모드로 들어간 다음, 버튼을 눌러 수정하세요. 길게 누르면 다른 자리로 끌어 옮길 수 있습니다. 버튼마다 레이블, 아이콘, 색, 그리고 동작이 하나씩 있습니다.

단계 순서는 한 줄에 한 단계입니다: key ctrl+s, text Hello, delay 500, launch notepad, url https://…, cmd …. 편집기의 테스트 버튼은 저장하지 않고 동작을 한 번만 실행합니다. 손이 많이 가는 단축키도 확정하기 전에 제대로 맞춰 볼 수 있습니다.

아이콘

이모지를 고르거나, 앱의 진짜 아이콘을 쓰세요

편집기에 아이콘 격자가 있어 이모지를 손으로 입력할 필요가 없습니다. 프로그램이라면 더 나은 방법이 있습니다. 바탕 화면에서 바로 가기를 끌어다 놓으면 앱의 진짜 아이콘도 함께 따라옵니다.

바탕 화면

바탕 화면에서 바로 가기를 끌어다 놓기

알림 영역 아이콘에서 편집기를 열면, 같은 화면을 안에 담은 진짜 데스크톱 창이 나타납니다. 위쪽 띠에 바로 가기, 프로그램, 폴더, 파일을 놓아 보세요. 대상이 해석되고 앱 아이콘이 추출되며, 버튼이 곧바로 태블릿에 나타납니다.

편집기가 브라우저 탭이 아니라 네이티브 창인 데에는 분명한 이유가 있습니다. 브라우저는 끌어다 놓은 파일의 경로를 웹 페이지에 절대 넘기지 않고, 게다가 Chromium은 놓인 바로 가기를 그 대상으로 해석해 버립니다. 즉 브라우저는 위치를 아는 대신 프로그램 전체를 읽으려 들게 됩니다.

동기화

어느 쪽에서 고치든, 바로

태블릿과 데스크톱 편집기는 같은 덱을 두 번 보는 것입니다. PC에서 버튼을 더하면 새로 고치지 않아도 태블릿에 나타나고, 태블릿에서 페이지 이름을 바꾸면 편집기의 페이지 목록도 따라옵니다. 모든 변경은 곧바로 디스크에 기록되고 최근 서른 개 판본이 남습니다. 그래서 두 기기에서 동시에 고친 경우, 한쪽이 조용히 덮어써지는 대신 거부됩니다.

포인터

화면이 터치패드가 됩니다

버튼에 마우스 동작을 주면, 누르는 순간 태블릿 전체가 트랙패드가 됩니다. 두 개의 슬라이더로 포인터와 스크롤 속도를 정하고, 스크롤 방향은 익숙한 쪽으로 뒤집을 수 있습니다.

공간

밀어서 페이지 넘기기

버튼을 페이지로 묶으세요: 편집용 한 장, 게임용 한 장, 늘 들어가 있는 그 회의용 한 장. 덱의 아무 곳에서나 좌우로 밀면 페이지 사이를 오갑니다. 페이지 이름이 잠깐 떠서 어디에 왔는지 알려 줍니다. 편집 모드에서는 같은 동작이 버튼 순서 바꾸기가 되므로, 둘이 부딪칠 일은 없습니다. 타일을 다른 페이지로 옮기려면 그 페이지의 탭 위로 끌어다 놓으세요.

공간

페이지, 열, 그리고 떠 있는 버튼

편집 모드에서는 아래 막대로 페이지를 더하고 지우며, 격자가 몇 열을 쓸지 정합니다. 열 수는 지금 보고 있는 페이지에 딸린 것이라, 계기 페이지는 네 열, 단축키 페이지는 다섯 열로 둘 수 있습니다. 페이지 탭을 한 번 더 누르면 이름을 바꿉니다. 막대가 숨겨져 있어도 떠 있는 버튼으로 페이지·편집기·테마에 갈 수 있습니다. 화면에 늘 남아 있는 유일한 것인 만큼, 편한 자리 어디로든 끌어 옮길 수 있습니다.

계기

기계의 상태를 덱 위에 올려 두기

타일은 버튼 대신 계기가 될 수도 있습니다: 프로세서, 메모리, 그래픽, 디스크 공간, 디스크 움직임, 네트워크 처리량, 온도, 가동 시간, 그리고 시계. 계기는 버튼과 같은 격자에 놓이므로 페이지, 열, 화면 가득 채우기, 끌어서 순서 바꾸기가 모두 그대로 통합니다. 새로 익힐 것은 없습니다. 누르는 것이 아니라 읽는 것이어서, 색을 채운 타일이 아니라 표면에 박아 넣은 계기처럼 그려집니다.

온도는 서버를 관리자 권한으로 실행해야 하고, 아예 센서를 내주지 않는 메인보드도 있습니다. 계기가 둘 중 어느 쪽인지 알려 줍니다. 하나는 당신이 고칠 수 있고, 다른 하나는 그럴 수 없기 때문입니다.

계기

버튼을 더하듯 똑같이 더하면 됩니다

편집 모드에서 타일을 하나 더하고 그 동작을 시스템 모니터로 지정하세요. 무엇을 잴지, 다이얼로 그릴지 막대로 그릴지, 그리고 색을 고릅니다. 아래 미리보기는 살아 있어서 저장하기 전에 실제 값이 보입니다. 레이블을 비워 두면 계기가 알아서 이름을 답니다.

측정은 계기가 있는 페이지가 열려 있는 동안에만 돌고, 마지막으로 보던 사람이 떠나면 멈춥니다. 그래서 계기가 없는 덱은 아무 부담도 주지 않습니다.

겉모습

열한 가지 테마, 버튼도 따라갑니다

대부분의 테마는 개발자에게 익숙한 색 구성에서 가져왔습니다: Nord, Gruvbox, Solarized, Catppuccin Mocha, Rosé Pine. 여기에 일부러 두 가지 색만 쓴 터미널풍 두 가지와 중립적인 밝은·어두운 테마가 더해집니다. 테마를 바꾸면 버튼 색이 색상을 유지한 채 새 팔레트로 옮겨집니다. 파란 버튼은 새 테마의 파랑이 되고, 배치는 그대로 읽힙니다.

패널은 결이 다릅니다. 팔레트가 아니라 방식 자체를 바꿔, 버튼을 계기와 같은 계기판 면으로 만들고 색은 위쪽 가장자리의 띠 하나로 줄입니다. 계기에도 따로 테마를 줄 수 있어서, 버튼은 한 테마를 따르고 계기는 다른 테마를 지키게 할 수 있습니다. 덱이 온통 한 가지 색이 되어 버렸다면, 테마 팔레트로 분배가 테마의 색을 버튼들에 나눠 줍니다. 모두 강조 색으로 칠하기는 다시 하나의 색으로 되돌립니다. 어떤 버튼이든 색 선택기에서 원하는 색을 따로 줄 수 있고, 막대와 떠 있는 버튼이 쓰는 강조 색도 직접 고릅니다. 어떤 색을 고르든 버튼 글자는 검정과 하양 사이에서 알아서 바뀝니다.

화면

화면을 남김없이 쓰기

버튼을 화면 가득 채우기를 켜면 행이 전체 높이를 나눠 가져, 두 줄짜리 버튼도 아래를 비우지 않고 화면을 채웁니다. 위아래 막대 자동 숨기기는 그것들을 완전히 없애고 버튼만 남깁니다. 떠 있는 버튼으로 되돌리거나, 빈 곳을 3초 길게 누르세요.

태블릿은 잠들지 않도록도 설정되어 있습니다. 덱이 열려 있는 동안 화면은 꺼지지 않습니다.

언어

스무 개 언어, 자동으로 선택

화면은 영어, 터키어, 스페인어, 프랑스어, 독일어, 이탈리아어, 포르투갈어, 러시아어, 우크라이나어, 폴란드어, 아랍어, 페르시아어, 우르두어, 중국어, 일본어, 한국어, 힌디어, 벵골어, 인도네시아어, 베트남어로 제공됩니다. 직접 고르지 않으면 브라우저 언어를 따르고, 아랍어·페르시아어·우르두어는 오른쪽에서 왼쪽 배치로 바뀝니다. 설정은 기기별이라 태블릿과 데스크톱 편집기가 서로 달라도 됩니다.

알아두면 좋은 것

방화벽
Windows가 처음에 연결 허용을 물을 수 있습니다. 태블릿이 서버에 닿지 않으면 개인 네트워크에 대해 허용하세요.
VPN
일부 VPN 클라이언트는 로컬 네트워크 트래픽을 막습니다. 아무것도 연결되지 않으면 VPN을 끄고 시도해 보세요.
관리자 창
관리자 권한으로 실행 중인 프로그램은 일반 프로그램의 입력을 받지 않습니다. 단축키가 그런 창에도 닿아야 한다면 서버를 관리자 권한으로 실행하세요.
페어링 코드
네트워크에서 코드를 가진 사람은 명령 실행을 포함해 PC를 조작할 수 있습니다. 코드는 본인만 알고 있고, 신뢰하지 않는 네트워크에서는 띄우지 마세요.
